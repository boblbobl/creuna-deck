Kanban

Go through the Kanban process

  • Use iterations as our Backlog
  • (Almost) all tasks should contain the following tasks:
    1. QA
    2. Deploy to Staging
    3. Deploy to Production
    4. Documentation (if required)
  • Use the following naming conventions:
    • Development tasks must be pre-fixed with "DEV:"
    • Configuration tasks must be pre-fixed with "CONF:"
    • ...
  • Defects must be created as Defects
  • Focus on compleness through WIP limits
  • Two boards: Daily - and Sign-off Board
  • US Owner will mark tasks as ready
